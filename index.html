3DX–SAP Integration & Migration
Proven Integration Patterns

Real project experience across all major SAP integration approaches — both on-premise and cloud, integration and data migration.

Integration

3DX → SAP S/4HANA via PFI & Event Publishing

Event-driven EBOM, Change Action, MFI, and Manufacturer data exchange triggered on 3DX Change Action completion. Custom web services on both sides with acknowledgement loop, middleware for payload transformation and routing.

PFIREST APICustom Web ServicesKafkaActiveMQ
Integration

3DX → SAP ECC via T Systems Connector

Classic ENOVIA–SAP integration using T Systems connector via RFC/BAPI. Covers EC Parts, ENOVIA EBOM and MBOM data exchange on Change Action completion. Connector customized to suit client-specific business cases and data models.

T Systems ConnectorRFC / BAPIEBOM / MBOMEC Part
Migration

SAP PM (ECC) → 3DEXPERIENCE via SAPJCo

Data migration from SAP Plant Maintenance into 3DEXPERIENCE. SAP Java Connector used to authenticate and call SAP PM BAPIs for equipment data extraction, converted to XPDMXML format and imported into 3DX.

SAPJCoSAP PM BAPIsXPDMXMLJava
PoC

3DX → SAP BTP via PFI + SAP Event Mesh + CPI

Cloud-native integration proof-of-concept connecting 3DX on-premise to SAP BTP. PFI publishes events into SAP Event Mesh, with SAP CPI acting as iPaaS middleware to orchestrate BOM data synchronisation into S/4HANA Public Cloud.
